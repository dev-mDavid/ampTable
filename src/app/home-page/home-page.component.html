Challenges

Aspects of this project I found fun & insightful

GET Request Data + Column State Data Configuration

From the onset, I knew data management was going to be the crux of the challenge (between HTTP Requests and NgRx State Management).
In the spirit of making re-usable components, container-to-presentational dataflow component model.

Drag & Drop Columns (without table)

A part of this challenge was explicitly specified to use & explore Angular's Drag & Drop CDK. And to not use tables that have sorting or resizing built-in. I opted to build my "table" and columns all from using the Drag & Drop.

Connect with ClickUp's API

One of the challenge's requirements was to get data via Http request. But it didn't specify what API source to use. I thought it would be a good learning experience to connect with ClickUp's API.

Column Width-Resize

There are multiple solutions for resizing components, including libraries. What I didn't like about them is that they all directly modified the DOM (which isn't considered "best practices"). Trying to do it "ng way", I learned more about and used @HostListener for DOM events.

Batch-Save Sorted Columns State

Conceptually, I understand that I want to save the appeared sorted-state of the columns to the store. But I couldn't figure out the configuration for updateMany() method for NgRx. After hours of research and trial-and-error, I pinned this to-do to the end of my priorities.
